# 서비스 피드백 분석

## 워크플로우

피드백 분석 시 체크리스트:
- [ ] 1. 피드백 내용 파악 및 우선순위 분류
- [ ] 2. 관련 코드 위치 탐색
- [ ] 3. 원인 분석 및 해결방안 도출
- [ ] 4. 리포트 파일 작성

## 우선순위 기준

| 우선순위 | 설명 |
|----------|------|
| P0 | 서비스 이용 불가, 핵심 기능 오류 |
| P1 | 사용성 저하, UX 문제 |
| P2 | 개선 제안, 선택적 기능 |

## 리포트 파일

**파일명**: `agent-guide/reviews/service-feedback_YYMMDD.md`

**구조**:
1. 제목 + 설명
2. 피드백 목록 테이블 (우선순위, 상태)
3. 상세 분석 & 해결방안
4. 작업 권장 순서
5. 검증 항목

## 상세 분석 형식

각 피드백 항목:

```markdown
### N) [피드백 제목]
- **증상**: 사용자가 겪는 문제
- **원인 추정**: 코드/설계 관점 분석
- **해결방안(권장)**: 구체적 수정 방향
- **영향 파일**: 수정 필요 파일 목록
- **코드 수정 가이드**: 현재 코드 vs 수정 방안
- **검증**: 수정 후 확인 방법
```

## 체크리스트

### 분석 시 확인 항목

- [ ] 피드백이 재현 가능한지 확인
- [ ] 관련 코드 파일/라인 특정
- [ ] 영향 범위 파악 (다른 기능에 미치는 영향)
- [ ] 해결방안의 사이드이펙트 검토

### 리포트 작성 후

- [ ] 우선순위 기준에 맞게 분류되었는지 확인
- [ ] 모든 피드백에 영향 파일 명시
- [ ] 해결방안에 코드 예시 포함
- [ ] 작업 권장 순서가 우선순위 반영
