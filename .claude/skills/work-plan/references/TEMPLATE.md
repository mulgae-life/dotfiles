# 단일 파일 계획서 템플릿

Lite / Standard 모드용. 규모에 맞는 섹션만 사용한다.

> 섹션 헤더 `(필수)`/`(선택)` 태그 준수. `(선택)` 섹션은 해당 없으면 섹션 자체를 생략.
> 변경 예시는 **핵심 시그니처/구조만** 작성 (전체 구현 X).

---

## Lite 모드 (파일 1-2개)

```markdown
# 작업 계획서: [작업명]

> 작성일: YYYY-MM-DD | 상태: 초안

## TL;DR (필수, 3줄)
- **무엇 / 왜**: [1-2문장]
- **범위**: 파일 N개 · 예상 N시간

## 개요

### 목표 (필수)
[1-2문장]

### 성공 기준 (필수)
- [ ] 기준 1
- [ ] 기준 2

### 범위 (필수)
- **포함**: [구체적 항목]
- **제외**: [명시적으로 빠지는 항목]

## 작업 목록 (필수)
- [ ] 작업 1
- [ ] 작업 2

## 검증 (필수)
\`\`\`bash
<검증 명령어 + 기대 결과>
\`\`\`

---

**이 계획에 대해 의견이 있으시면 말씀해 주세요.**
```

import { z } from "zod";

export const UserSchema = z.object({
  id: z.string().uuid(),
  email: z.string().email(),
  role: z.enum(["admin", "member"]),
});

export type User = z.infer<typeof UserSchema>;
