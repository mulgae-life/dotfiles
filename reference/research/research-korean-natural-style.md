# 조사: LLM 한국어 응답의 탈AI 문체 — 규칙 나열을 넘는 접근 (2026-07-17)

> deep-research 워크플로 실행 결과. 검색 각도 5개, 출처 21건 확보, 주장 97건 추출 → 상위 25건을
> 3표 적대적 검증(2/3 반박 시 기각) → 22건 확증, 3건 기각. 에이전트 103개 투입.
> 배경: `.claude/rules/communication.md` 문체 규칙과 work-verify `style-check.md` 점검표를
> 운영 중이나 결과가 만족스럽지 않아, 기존 규칙과 겹치지 않는 새 기법을 발굴하는 것이 목표.

## 핵심 결론 — 세 층위

**1층. 생성 시점 규칙이 아니라 후처리 리라이트 패스가 유망한 구조다.**
한국어 특화 im-not-ai(탐지→수술적 윤문→자체검증), 영어권 stop-slop(5축 자기 채점 + few-shot 예문),
토스의 단계별 GPT 리뷰 봇이 전부 같은 구조를 독립적으로 채택했다. "하지 마라" 규칙을 컨텍스트에
쌓는 방식은 생성 중 주의가 분산되면 무시되지만, 완성문을 대상으로 한 별도 패스는 점검이 전담 작업이 된다.
단, 실증된 것은 독립 구현 3종의 수렴 채택까지로, 효과의 독립 벤치마크는 아직 없다.

**2층. 한국어 AI 문체는 계측 가능한 마커로 환원된다.**
연결어미 뒤 쉼표 과다(인간 4.1~13.3% vs LLM 15.6~28.0%), 띄어쓰기 과규칙성(의존명사 띄어쓰기
비율 SD 0.02), 체언 과다·용언/어미 부족 — ACL 2025 동료심사 논문의 정량 수치다. 감이 아니라
숫자로 점검할 수 있다.

**3층. 검증은 루브릭 + 타 계열 판정자.**
직관 판정 60% → 루브릭 판정 90%(파일럿). Claude가 Claude 출력을 자가 채점하는 구조는
자기선호 편향 위험이 실증됐고, 다른 계열 판정자 앙상블이 완화책이다(완전 제거는 불가).

또한 문체 표층과 별개로 **내용 층위**(추상론 중심·개인 경험/감정/단정 배제)가 AI 인상의 독립
축으로 지목됐다(한국어 직접 근거는 무하유 1건 — F6 medium) — 문체 규칙만으로는 해결되지 않는다.

## 확증 파인딩 11건

### F1. im-not-ai (Humanize KR) — 한국어 특화 후처리 리라이트 패스 [high, 3-0]

- 출처: https://github.com/epoko77-ai/im-not-ai (3.8k 스타, v2.0.0 2026-05, Claude Code 플러그인 + Codex CLI 배포)
- 규칙을 생성 시점에 주입하지 않고, 완성문을 10대 카테고리 × 71개 서브 패턴(심각도 S1/S2/S3)으로
  스팬 단위 탐지한 뒤 탐지 스팬에만 수술적으로 윤문하고 자체검증.
- 과윤문 가드레일: 변경률 30% 초과 시 경고, 50% 초과 시 강제 중단. 사실·수치·고유명사·직접 인용 100% 보존.
- Strict Mode 파이프라인: ai-tell-detector → korean-style-rewriter → 병렬 검증 팀. agents/ 62개 파일이 실구현.
- 한정: 가드레일은 프롬프트 수준 정책(기술적 강제 아님). 효과의 독립 벤치마크 없음.

### F2. im-not-ai 탐지 택소노미 — 이식 가능한 신규 패턴 목록 [high, 3-0]

- 출처: 위 레포 `.claude/skills/humanize-korean/references/ai-tell-taxonomy.md` (587줄, v2.0)
- 기존 규칙(communication.md·style-check.md)에 없는 패턴: 이중 조사('~에서의/~으로의'),
  강제 대명사('그/그녀'), AI 관용구('시사하는 바가 크다', '주목할 만하다'), 기계적 구조
  ('첫째/둘째/셋째'), 문장 길이 표준편차 낮음(단조 리듬의 정량화), 서식 임계값(따옴표 5회 초과 등).
- 정규식/임계값 수준으로 목록화되어 그대로 이식 가능. 일부 패턴은 저자 단독 관찰('사용자 관찰' 라벨)로 학술 검증 전.

### F3. stop-slop — 자기 채점 루브릭 + few-shot 예문 병행 구조 [high, 3-0]

- 출처: https://github.com/hardikpandya/stop-slop
- 메커니즘 ①: 초안을 Directness·Rhythm·Trust·Authenticity·Density 5축 각 1-10점으로 자기 채점,
  합계 35/50 미만이면 재작성 강제("Below 35/50: revise." 축자 확인).
- 메커니즘 ②: 금지어 목록(phrases.md)과 **별도로** before/after 변환 예문 파일(examples.md)을
  두는 few-shot 병행 구조. 구조적 클리셰 파일(structures.md)도 분리.
- 채점은 인컨텍스트 자기 채점이며 프로그램 루프는 아님.

### F4. 국립국어원 새국어생활 (김순영 2012) — 번역투 4대 패턴의 학술 근거 [high, 3-0 × 4]

- 출처: https://www.korean.go.kr/nkview/nklife/2012_1/22_0105.pdf (PDF 원문 직접 추출 검증)
- (1) have 직역 '가지다' → 존재·상태 서술('아름다운 목소리를 가지고 있다'→'목소리가 아름답다')
- (2) '~에 의해' 행위자 표지 피동 → 능동 전환 **또는 조사 교체**('한테/에게/으로')
- (3) 전치사구 직역('~에 관하여/~로부터/~를 통해') → 문맥별 치환('에게서/~에서 벗어나는/~으로')
- (4) 복수 '-s'의 기계적 '-들' → **부사로 복수 의미 표현**('아이디어들과 방법들'→'여러 가지 남다른 생각과 방법')
- 기존 규칙 대비: 패턴 자체는 style-check.md와 상당 부분 겹침(가지다·피동·~들). 신규 가치는
  **대안 제시 방식**(조사 교체, 부사화)과 학술 전거. 한글문화연대·카카오 가이드가 교차 확인.

### F5. 토스 테크니컬 라이팅 — 명사화 해체 + 단계별 리뷰 봇 [high, 3-0 × 3]

- 출처: https://technical-writing.dev (github.com/toss/technical-writing raw 소스 축자 확인)
- 명사화 해체: '시스템 모니터링 수행을 통해 서버 성능 측정 작업을 실시하게 됩니다'
  → '시스템을 모니터링해서 서버 성능을 측정합니다'
- 군더더기 한자어: '수행하다·실행하다·진행하다'는 빼도 의미 전달('로그 파일을 삭제하는 작업을
  수행합니다'→'로그 파일을 삭제합니다') — LLM 한국어의 전형적 군더더기와 정확히 겹침
- 운영 방법론: 문장 원칙 5종(주체 명확화[하위: 능동형]·간결·구체성·자연스러운 한국어 표현·일관성)을 규칙 나열이 아닌
  **단계별 GPT 리뷰 봇**(문서 유형→정보 구조→문장 다듬기, "스텝별 봇을 만들어야 답변이 잘 나와요")으로 운영.
  개선 옵션을 나열하지 않고 '모두 반영한 하나의 좋은 안'을 출력하도록 설계.

### F6. 내용 층위 — 추상론·무경험이 AI 인상의 독립 축 [high 3-0 → medium 하향: 한국어 직접 근거는 무하유 1건]

- 출처: 무하유 GPT킬러 매뉴얼(manual.muhayu.com/montly-gpt-killer-labs/detected-ai-patterns,
  국내 대학 96% 배포 탐지 서비스 — 유일한 한국어 직접 근거) + arXiv 2510.05136(언어 특성 서베이,
  "heavily concentrated on English data" 자인) + arXiv 2606.04177(284개 언어 특성 실증, 영어 MAGE
  벤치마크 전용). arXiv 2건은 영어 기준 방증이라 한국어 교차 검증으로 세지 않는다. Nature HSSC 2025
  (개인 대명사 감소)·ACM WebSci 2026(페르소나 프롬프팅)은 검증 시점 식별자 미보존 — 재검증 불가
- AI 한국어 글은 개념·원리·방법론 등 추상적·일반론 중심, 개인 경험·감정·확신·주관적 통찰 배제 경향.
- 역으로 구체 경험·감정·단정을 넣으면 AI 인상 감소. 페르소나 프롬프팅으로 1인칭 주관 서술을 만들 수
  있다는 보고(ACM WebSci 2026)는 식별자 미보존으로 재검증 불가 — 미검증 보고로만 취급.
- 함의: 문체(표층) 규칙만으로 해결 안 되는 축. "일반론 대신 이 프로젝트의 구체 사실·실측·단정으로 말하라"가 대응.

### F7. KatFishNet — 한국어 AI 텍스트의 정량 마커 [high, 3-0 × 4]

- 출처: ACL 2025 Main (arXiv 2503.00032) + 후속 전문가 패널 연구(arXiv 2601.19913)
- (1) 쉼표 과다: 에세이 쉼표 포함율 인간 26.3% vs LLM 61.0%. 특히 **연결어미 뒤 쉼표**
  ("~하고, ~해서,") 인간 4.1~13.3% vs LLM 15.6~28.0% — 영어 구두점 관습 내재화 추정
- (2) 띄어쓰기 과규칙성: LLM은 맞춤법을 경직되게 준수(의존명사 띄어쓰기 SD 0.02), 인간은 문체적으로 생략
- (3) 품사: 인간은 어미·용언 풍부, LLM은 체언(명사) 중심 — 토스 명사화 해체와 정합
- (4) 레지스터 항상성·번역투 미세 패턴. 후속 연구에서 인간 패널이 동일 마커로 90~100% 탐지.
- 한정: 검증 장르는 에세이/시/초록(어시스턴트 대화체 아님), 생성기는 GPT-4o·Solar·Qwen2·Llama3.1로
  **Claude 미포함**. 품사 마커는 장르 의존(에세이 AUROC 83.0% vs 초록 53.3%).

### F8. 루브릭 기반 판정 — 직관 60% → 루브릭 90% [medium, 2-1]

- 출처: arXiv 2601.19913 (Park & Han, 연세대, 2026-01, 프리프린트)
- 한국어문학 전공 학부생 3인 패널, 3단계 평가(§5.1): Phase 1 직관 30편 → Phase 2 한국어 특화
  루브릭(LREAD) 별도 30편 → Phase 3 정답 재공개 후 초등 에세이 10편. 다수결 정확도 60%→90%→100%,
  Fleiss' κ -0.09→0.24→0.82 — κ 0.82는 루브릭 전환이 아니라 Phase 3 수치다.
- 한정 다수: 소규모 파일럿, 정답 공개가 단계 사이에 끼어 루브릭 단독 인과 미분리, 동료심사 전.
- 함의(방향성 근거로만): work-verify 문체 점검을 체크리스트("있나 없나")에서 항목별 루브릭
  채점("몇 점인가, 임계 미달 시 재작성")으로 격상.

### F9. 모델별 한국어 시그니처 차이 [medium, 3-0이나 일반화 한계]

- 출처: arXiv 2601.19913 §6.3/Appendix D.2
- Qwen2: 한자어 명사화·직역투(calque). GPT-4o: 수식어 과다 명사구·잦은 대명사 지시 등 영어식 번역투.
- **Claude는 생성기로 검증된 적 없음** — Claude Code 적용은 재검증 필요. 논문도 '계열 수준 가설'로 한정.

### F10. LLM-judge 자기선호 편향과 앙상블 완화 [high, 3-0]

- 출처: arXiv 2604.06996 + 2604.22891 교차
- "Across the board, ensembling leads to the same or higher accuracy and lower SPB, though it
  does not fully eliminate SPB." — 5개 상이 계열 판정자 다수결이 자기선호 편향을 일관되게 낮춤.
- 독립 연구에서 앙상블 패널로 SPB 평균 50%+ 감소(GPT-4o 82%→30%).
- 함의: Claude 단독으로 Claude 한국어 출력을 채점(현 verifier/work-verify 구조)하면 편향 위험.
  Codex 등 타 계열 교차 판정(recursive-discussion 스킬 활용)이 구조적으로 유리.

### F11. 'It's not X — it's Y' 부정 병렬 — 가장 끈질긴 AI 징후 [medium]

- 출처: tropes.fyi 원문 + GeekNews 번역(news.hada.io/topic?id=27321) + The Atlantic 2026 교차
- 영어권에서 가장 흔하게 식별되는 AI 문체 징후. 인간 대비 약 3배 빈도, 'delve' 등 다른 틱이
  교정된 뒤에도 전 주요 모델에 잔존하는 가장 끈질긴 특징(The Atlantic).
- 한국어 '~이 아니라 ~이다'로 1:1 이식 가능하되, 원문 스스로 "한 편에 1회는 무방" —
  전면 금지가 아닌 빈도 차단으로 도입해야 함. (기존 style-check.md 1.2가 이미 같은 빈도 기준
  "한 문서에 한 번"을 보유 — 추가 이식 불요)

## 한계·공백

- **Claude의 한국어 시그니처가 정확히 연구 공백**: 기존 연구는 GPT-4o·Qwen2 등만 생성기로

- 조사 축 (1) 한국어 커뮤니티 실사용 팁은 GeekNews·im-not-ai 외 클리앙·디시·velog·유튜브발

   한국어 문체 스킬을 만들거나 im-not-ai 플러그인 자체를 평가. 변경률 가드레일 포함.
2. **[규칙→예문] few-shot 예문 파일 신설** — stop-slop의 examples.md 구조처럼 금지 목록과
   별도로 before/after 변환 예문을 두고 work-verify·communication.md에서 참조.
3. **[신규 마커 이식] style-check.md 증보** — 연결어미 뒤 쉼표, 체언 과다(명사화), 이중 조사,
   '첫째/둘째' 기계 구조, 내용 층위(일반론 대신 구체 사실·단정). '~이 아니라 ~이다' 빈도 기준은
   기존 1.2에 이미 있어 제외.
4. **[검증 격상] work-verify를 루브릭 채점형으로** — stop-slop 5축의 한국어판(직접성·리듬·
   신뢰·진정성·밀도) 1-10점 채점 + 임계 미달 시 재작성.
5. **[교차 판정] Codex로 Claude 출력 문체 검증** — recursive-discussion 스킬 활용, 자기선호
   편향 회피.

   분포)로 Claude Code 실제 응답을 측정, 도입 전후 비교.
